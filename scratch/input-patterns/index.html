Input Patterns

inputcontrols

Build reliable keyboard and mouse controls that feel responsive.

Milestones

  1. Map keys: Choose clear controls and document them on screen.
  2. Hold vs tap: Use forever loops for hold controls, events for taps.
  3. Debounce: Add short waits to prevent accidental repeats.

Guide

Goal

Create control schemes that match your game or story.

Try it

Build left/right movement with a loop that checks keys.

forever
  if <key [left arrow v] pressed?> then
    change x by (-5)
  end
  if <key [right arrow v] pressed?> then
    change x by (5)
  end
end

Checkpoint

You can explain the difference between tap and hold controls.

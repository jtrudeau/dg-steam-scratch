Broadcast + Receive Patterns

broadcasteventscoordination

Coordinate sprites and scenes using named messages so your story stays in sync.

  • Name messages clearly: Use labels like `start-level`, `win`, `game-over`, or `scene-2` instead of generic names.
  • One source of truth: Keep scene changes in one script that broadcasts; other sprites just listen and react.
  • Chain sequences: Broadcast from within a listener to trigger the next step (e.g., intro → gameplay → outro).
  • Handle duplicates: If two things fire at once, add small waits (0.2s) or separate messages to avoid clashes.
  • Debug with logs: Temporarily `say` the message name inside each listener to verify everyone hears it.

Debug Like a Scientist

debuggingeventslogic

A quick protocol for hunting bugs: observe, predict, isolate, and retest with intention.

  • Pause and observe: Click the green flag and list what you expected vs. what actually happened before touching any code.
  • Isolate one script: Right-click → duplicate a sprite or disable one script at a time to see if the bug follows.
  • Add print-style clues: Drop `say` blocks or temporary color changes inside loops so you know which blocks are running.
  • Check events and resets: Make sure green flag scripts set starting positions/variables and broadcasts match exactly.
  • Retest and log: After each tweak, retest and jot one sentence in your debugging log so you can roll back or teach peers.

Make Your First Sprite Say Hello

spriteslooksevents

Kid-friendly walkthrough for adding a sprite, dressing it up, and making it talk when the green flag waves.

  • Pick a buddy: Click the blue cat button to add a sprite. Choose any character that makes you smile, or draw your own in the Paint option.
  • Give them style: Hop into the Costumes tab and color in outfits or switch costumes. Think superhero capes, lab coats, or SDG flair.
  • Teach a greeting: In the Code tab, drag `when green flag clicked` and snap a `say [Hello DG!] for (2) seconds` block underneath. Your sprite now talks when the flag starts the story.
  • Add a wiggle: Slide in `move (10) steps` and `move (-10) steps` blocks to create a tiny dance. You can add a `wait (0.5) seconds` in between for rhythm.
  • Show it off: Click the green flag and watch your sprite wave hello. Ready for more? Add a key press block so it tells a joke when you press space!

Scratch Interface Tour

spritesstageevents

A five-stop walkthrough of the Scratch editor so new makers know where to click and how to debug.

  • Stage (right side): This is where your project runs. Try the green flag/red stop buttons while narrating what should happen.
  • Sprites list: Every character or object shows up here. Click a sprite to edit its code and costumes.
  • Blocks palette: Color-coded categories (Motion, Looks, Sound, Events, Control, Sensing, Operators, Variables, My Blocks) live here.
  • Scripts area: Drag blocks into this blank canvas and snap them vertically. Wrap behaviors with loops or conditionals as needed.
  • Code / Costumes / Sounds tabs: Use Costumes to edit art and Sounds to add effects or narration so projects feel complete.

Scratch Setup Walkthrough (Session 01)

accountsonboardingevents

Visual step-by-step for creating EMSB-backed Scratch accounts, verifying email, and shipping the first Hello DG animation.

  • Join and set username: Homepage → Join → pick a school-safe handle (no real names), e.g., dg-rocket-17.
  • Password + EMSB email: Follow the program password pattern and enter the EMSB email carefully.
  • Verify: Find the Scratch email in EMSB inbox, click Confirm, and sign back in.
  • Ship first animation: Create project, add sprite/backdrop, build Hello DG stack, add a key-press effect, rename and save.
  • Need pictures?: See /resources/setup for annotated screenshots and downloadable checklist.

Variables 101

variablesoperatorsscoring

Create, set, and change variables to track score, timers, and state reliably.

  • Name with purpose: Use clear names like `score`, `timer`, or `scene` so teammates know what changes it.
  • Always initialize: Under the green flag, `set` variables to starting values before any gameplay begins.
  • Show or hide on stage: Toggle the checkbox for HUD values or hide them when you switch to cutscene backdrops.
  • Combine with operators: Use `change` plus `if` checks to award points, track lives, or count completed tasks.
  • Reset on retry: Add a broadcast (e.g., `restart`) that resets variables and positions so users can replay.

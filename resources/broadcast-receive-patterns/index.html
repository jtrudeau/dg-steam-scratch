Broadcast + Receive Patterns

broadcasteventscoordination

Coordinate sprites and scenes using named messages so your story stays in sync.

Steps

  1. Name messages clearly: Use labels like `start-level`, `win`, `game-over`, or `scene-2` instead of generic names.
  2. One source of truth: Keep scene changes in one script that broadcasts; other sprites just listen and react.
  3. Chain sequences: Broadcast from within a listener to trigger the next step (e.g., intro → gameplay → outro).
  4. Handle duplicates: If two things fire at once, add small waits (0.2s) or separate messages to avoid clashes.
  5. Debug with logs: Temporarily `say` the message name inside each listener to verify everyone hears it.
